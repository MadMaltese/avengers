# Database - NFR 요구사항

## 성능
| 항목 | 요구사항 |
|------|----------|
| 커넥션 풀 | HikariCP, max 10 connections (소규모 단일 매장) |
| 쿼리 응답 | 단일 쿼리 50ms 이내 |
| 인덱스 | 주요 조회 패턴에 인덱스 적용 (세션별, 테이블별, 날짜별) |

## 데이터 무결성
| 항목 | 요구사항 |
|------|----------|
| FK 제약조건 | 모든 관계에 FK 적용 |
| CHECK 제약조건 | price >= 0, quantity > 0 |
| UNIQUE 제약조건 | store.code, (store_id, table_no) |
| 트랜잭션 | 이용 완료(Order→OrderHistory 이동) 시 단일 트랜잭션 보장 |

## 유지보수성
| 항목 | 요구사항 |
|------|----------|
| 마이그레이션 | 버전 관리된 SQL 스크립트 |
| 시드 데이터 | 개발/테스트용 초기 데이터 제공 |
