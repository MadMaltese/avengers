# Backend - NFR 요구사항

## 성능
| 항목 | 요구사항 |
|------|----------|
| API 응답 시간 | 200ms 이내 (p95) |
| SSE 이벤트 전달 | 주문 발생 후 2초 이내 |
| 동시 접속 | 20 테이블 + 관리자 1명 동시 처리 |
| SSE 연결 | 최대 25개 동시 SSE 연결 유지 |

## 보안
| 항목 | 요구사항 |
|------|----------|
| 인증 | JWT 토큰 (16시간 만료) |
| 비밀번호 | bcrypt 해싱 (strength 10) |
| 로그인 제한 | 10회 실패 시 30분 잠금 |
| CORS | Frontend 도메인만 허용 |
| API 보호 | JWT 필터로 인증 필수 (로그인 API 제외) |
| 역할 기반 접근 | TABLE 역할: 고객 API만, ADMIN 역할: 관리자 API만 |

## 가용성
| 항목 | 요구사항 |
|------|----------|
| 서버 가동 | 매장 영업 시간 동안 안정 운영 |
| SSE 재연결 | 연결 끊김 시 클라이언트 자동 재연결 |
| 에러 처리 | GlobalExceptionHandler로 일관된 에러 응답 |

## 유지보수성
| 항목 | 요구사항 |
|------|----------|
| 로깅 | SLF4J + Logback, 요청/응답 로깅 |
| 코드 구조 | Layered Architecture 준수 |
| API 문서 | Swagger/OpenAPI 자동 생성 |
