# Frontend - 비즈니스 규칙

## 클라이언트 검증 규칙

| 규칙 ID | 화면 | 규칙 |
|---------|------|------|
| BR-FE01 | 장바구니 | 수량 최소 1 (0 이하 시 항목 삭제) |
| BR-FE02 | 장바구니 | 빈 장바구니로 주문 불가 |
| BR-FE03 | 관리자 로그인 | 매장 코드, 사용자명, 비밀번호 모두 필수 |
| BR-FE04 | 테이블 설정 | 매장 코드, 테이블 번호, 비밀번호 모두 필수 |
| BR-FE05 | 메뉴 등록 | 메뉴명 필수, 가격 >= 0, 카테고리 선택 필수 |
| BR-FE06 | 주문 내역 | 현재 세션 주문만 표시 (authStore.sessionId 기준) |

## UI 규칙

| 규칙 ID | 규칙 | 설명 |
|---------|------|------|
| BR-UI01 | 터치 타겟 최소 44x44px | 모든 버튼/터치 영역 |
| BR-UI02 | 주문 성공 후 메뉴 화면 리다이렉트 | 주문 번호 표시 후 자동 이동 |
| BR-UI03 | 삭제/이용완료 시 확인 팝업 | 파괴적 작업 전 확인 필수 |
| BR-UI04 | 신규 주문 시각적 강조 | 색상 변경 + 애니메이션 |
| BR-UI05 | SSE 연결 끊김 시 자동 재연결 | 네트워크 복구 시 재연결 시도 |

## 장바구니 로직

| 규칙 | 설명 |
|------|------|
| localStorage persist | Zustand persist middleware 사용 |
| 서버 전송 시점 | 주문 확정 버튼 클릭 시에만 |
| 새로고침 유지 | localStorage에서 복원 |
| 주문 성공 시 비우기 | clearCart() 호출 |
