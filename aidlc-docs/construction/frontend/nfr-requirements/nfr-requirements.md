# Frontend - NFR 요구사항

## 성능
| 항목 | 요구사항 |
|------|----------|
| 초기 로딩 | 3초 이내 (First Contentful Paint) |
| 페이지 전환 | 즉시 (SPA 클라이언트 라우팅) |
| 장바구니 조작 | 즉시 반응 (로컬 상태) |
| SSE 이벤트 반영 | 수신 즉시 UI 업데이트 |

## 사용성
| 항목 | 요구사항 |
|------|----------|
| 터치 타겟 | 최소 44x44px |
| 반응형 | 태블릿 해상도 최적화 (768px~1024px) |
| 시각적 피드백 | 모든 사용자 액션에 즉각 피드백 |
| 에러 표시 | 사용자 친화적 에러 메시지 |
| 로딩 상태 | API 호출 시 로딩 인디케이터 |

## 보안
| 항목 | 요구사항 |
|------|----------|
| 토큰 저장 | localStorage (JWT) |
| API 호출 | Authorization 헤더에 Bearer 토큰 자동 첨부 |
| 만료 처리 | 401 응답 시 로그인 화면 리다이렉트 |

## 유지보수성
| 항목 | 요구사항 |
|------|----------|
| 타입 안전성 | TypeScript strict 모드 |
| 코드 품질 | ESLint + Prettier |
| 컴포넌트 구조 | 페이지/컴포넌트/훅/스토어 분리 |
