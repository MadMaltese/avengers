# 테이블오더 서비스 - 페르소나

## Persona 1: 고객 (Customer)

| 항목 | 내용 |
|------|------|
| **이름** | 매장 방문 고객 |
| **역할** | 테이블에서 메뉴를 조회하고 주문하는 사용자 |
| **목표** | 대기 없이 빠르고 편리하게 음식을 주문하고, 주문 상태를 확인 |
| **기술 수준** | 다양함 (스마트폰/태블릿 기본 조작 가능) |
| **사용 환경** | 매장 내 테이블에 설치된 태블릿 (브라우저 기반 웹UI) |
| **핵심 니즈** | 직관적 UI, 빠른 주문 프로세스, 실시간 상태 확인 |
| **불편 사항** | 복잡한 로그인 절차, 느린 응답, 불명확한 주문 상태 |

### 사용 시나리오
1. 테이블에 앉으면 태블릿이 이미 로그인된 상태
2. 메뉴를 탐색하고 장바구니에 담음
3. 주문을 확정하고 주문 번호를 확인
4. 추가 주문이 필요하면 반복
5. 주문 내역에서 현재 주문 상태를 확인

## Persona 2: 매장 관리자 (Admin)

| 항목 | 내용 |
|------|------|
| **이름** | 매장 운영자 |
| **역할** | 매장의 주문, 테이블, 메뉴를 관리하는 사용자 |
| **목표** | 실시간으로 주문을 모니터링하고, 매장 운영을 효율적으로 관리 |
| **기술 수준** | 중간 (기본적인 웹 애플리케이션 사용 가능) |
| **사용 환경** | 카운터/사무실 PC 또는 태블릿 브라우저 |
| **핵심 니즈** | 실시간 주문 파악, 테이블 상태 관리, 메뉴 관리 |
| **불편 사항** | 주문 누락, 테이블 상태 혼동, 복잡한 관리 도구 |

### 사용 시나리오
1. 매장 오픈 시 관리자 로그인
2. 대시보드에서 실시간 주문 모니터링
3. 주문 상태를 변경 (대기중 → 준비중 → 완료)
4. 필요 시 주문 삭제 또는 테이블 이용 완료 처리
5. 메뉴 추가/수정/삭제 관리
